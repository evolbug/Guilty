Classes¶

There are 3 categories for Guilty's classes.

  • Primitives
  • Extensions
  • Composites

Their relations are simple, Primitives are the most basic components, providing base building blocks for Composites, i.e. the widgets themselves. Extensions are what give Composites their functionality, like making them responsive to clicks, text input, etc.

Below each class is described with properties, methods and events they can receive.

type property_name
This describes properties

Events

function love.<event>()
    widget:event(<event>)
end
This block describes events

:method( type name ) → nil

This is a method definition

The class descriptions show only differences from their parent class. To collect all functions and properties a class has, look at the class they extend.

Components¶

Component¶

Guilty is built on a component framework called Comfy, and thus all classes of Guilty possess these methods and properties:

class Component

Component() → self

table children
a list containing all attached children

Component parent
the component this one is attached to

:attach(Component ...) → ...

Attaches other child to self and collects it in children. The attached child's parent will be set to self.
This function is used most often to attach elements to one another, e.g. attach a Button to a Container.

Attaching child will override its variable parent.
A component should be attached to only one parent.

:detach(Component child) → nil

Detaches child from self, removing it from children.

:event(table events) → nil

Standard event propagation. Push events down to all children.
Events are handled first-come-first-serve.

:bubble(table events) → nil

Push-down-bubble event propagation. Push events down to all children but handle bottom-up.
Events are handled starting from the deepest child in the tree upwards.

:rise(table events) → nil

Rising event propagation. Push events up to parent.
Events are handled by parents, pushing them upwards with :rise() until the topmost parent is reached.

Primitives¶

Primitives are used to build more complex widgets, although they can be used on their own.

WidgetBase¶

WidgetBase is the base of all widget classes. It provides and manages the bare necessities of a widget.

class WidgetBase extends Component

WidgetBase(number x, number y, number w, number h) → self

number x, y
position relative to parent. Can have a special value 'center' which centers it in the parent.

number w, h
widget size

bool update
a flag that tells if a widget must be rerendered

bool visible = true
widget visibility flag

table theme = Theme(theme[<widget name>])
widget's theme. It's automatically found in the global theme table. See Theming

Events

widget:rise('update')
This event is used to signal that a branch of dependant widgets should be rerendered

:absolute() → number x, number y

returns absolute widget position on screen

:relative() → number x, number y

returns widget position relative to parent. Usually the position given on creation, or, if it's the special value 'center', will calculate the center of parent widget.

:draw() → nil

draws the widget with it's children

:colored(function func, ...) → nil

draws a love.graphics primitive with theme colors

Rectangle¶

Rectangle is a love.graphics.rectangle wrapped in WidgetBase for use as a widget component

class Rectangle extends WidgetBase

Rectangle(number x, number y, number w, number h) → self

Text¶

Text can be used on it's own as a label or formatted text. Colored text format ({color, string, ...}) shouldn't be used.

class Text extends WidgetBase

Text(number x, number y, string text) → self

string buffer = text
internal string buffer for easier manipulation. Shouldn't be manipulated directly

love.Text text
the actual text object. Shouldn't be manipulated directly

:refresh() → nil

refreshes text widget's size and position (y, w and h). Used internally, should be called each time text is manipulated

:set(string text, number wrap=parent.w, string align='left') → nil

set buffer to text and update the love.Text object

:add(string text, number wrap=parent.w, string align='left') → nil

add text to buffer and update the love.Text object

:remove(number a=#buffer-1, number b=#buffer) → nil

remove characters from buffer, starting from a to b, and update the love.Text object

:clear() → nil

erase buffer and reset love.Text object

Composites¶

Composites are widgets that are built from Primitives and Extensions. They must be attached to a Container to be rendered.

Container¶

The container is the base of all Composites. It's most basic use is a rendering window. Widgets must be :attach() -ed to a container, and the container must be :draw() -n.

class Container extends WidgetBase

Container(number x, number y, number w, number h) → self

Creates a new Container object that stores and renders attached widgets

love.Canvas canvas
attached widgets get rendered to this canvas

:draw() → nil

Renders the container with all attached widgets

function love.load()
  window = Container(10, 10, 200, 200)
end

function love.draw()
  window:draw()
end

Button¶

class Button extends Container

Button(number x, number y, number w, number h, string label) → self

Rectangle panel
Button background panel

Text text
Button label Text object

Clickable onclick
Clickable extension, makes button clickable

function love.mousepressed(x, y, button, istouch)
    Container:event{['mousepress']={x, y, button, istouch}}
end

function love.mousereleased(x, y, button, istouch)
    Container:event{['mouserelease']={x, y, button, istouch}}
end

function love.load()
    window = Container(0, 0, 100, 100)
    mybutton = window:attach(Button(10, 10, 100, 20, 'Button'))

    mybutton.onclick.primary = function(x, y) -- callback
        print("hello at x:"..tostring(x).." y:"..tostring(y))
    end
end

Checkbox¶

class Checkbox extends Container

Checkbox(number x, number y, number radius, string text, bool state) → self

Rectangle base
Checkbox base panel

Rectangle check
the "check" itself

Text text
checkbox's label

Clickable onclick
Clickable extension

bool state=false
current state of the checkbox

function love.mousepressed(x, y, button, istouch)
    Container:event{['mousepress']={x, y, button, istouch}}
end

:toggle() → nil

called when checkbox is clicked, toggles state. Do not override

function love.load()
    window = Container(0, 0, 100, 100)
    mycheck = window:attach(Checkbox(10, 10, 80, 20, 'Checkbox'))
end

TextBox¶

class TextBox extends Container

TextBox(number x, number y, number w, number h) → self

Rectangle base
TextBox base panel

Text text
Interact with this Primitive to set, add, remove text

function love.load()
    window = Container(0, 0, 100, 100)
    mytext = window:attach(TextBox(10, 10, 80, 80))

    mytext:add('hello world!\n')
end

TextBoxScrollable¶

class TextBoxScrollable extends TextBox

TextBoxScrollable(number x, number y, number w, number h) → self

Focus focus = Focus('mousemove')
Focuses on TextBox when mouse is hovering above it

Scroll scroller
Scroll extension, allows widget to be scrolled

ScrollBar scrollbar
ScrollBar extension, adds a scrollbar element based on scroller values

function love.mousemoved(x, y)
    Container:event{['mousemove']={x, y}}
end

function love.wheelmoved(x, y)
    Container:event{['wheelmove']={x, y}}
end

function love.load()
    window = Container(0, 0, 100, 100)
    mytext = window:attach(TextBoxScrollable(10, 10, 80, 20))

    mytext:set('imagine a lot of text here')
end

TextInput¶

class TextInput extends TextBox

TextInput(number x, number y, number w, number h) → self

Focus focus = Focus('mousepress')
Focuses on TextInput when mouse is clicked

string buffer
internal string buffer for easier manipulation. Shouldn't be manipulated directly

BlinkingCursor cursor
Cursor extension, blinks every .7 seconds

Input
Input extension, allows widget to receive keyboard input

function love.mousepressed(x, y, button, istouch)
    Container:event{['mousepress']={x, y, button, istouch}}
end

function love.keypressed(key)
    Container:event{['keypress']={key}}
end

function love.textinput(text)
    Container:event{['textinput']={text}}
end

function love.update(dt)
    Container:event{['delta']={dt}}
end

function love.load()
    love.keyboard.setKeyRepeat(true)

    window = Container(0, 0, 100, 100)
    myinput = window:attach(TextInput(10, 10, 80, 20))
end
